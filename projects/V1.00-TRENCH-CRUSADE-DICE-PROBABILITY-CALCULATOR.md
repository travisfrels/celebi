# V1.00 Trench Crusade Dice Probability Calculator

| Status | Set On |
|--------|--------|
| Planning | 2026-03-13 |
| Active | 2026-03-13 |

## Context

### Situation

The celebi repo is a new, empty repository with no source code. The user plays Trench Crusade (a tabletop miniatures wargame) and wants a desktop tool to calculate the probability of achieving each possible outcome under the game's core dice mechanic, so they can make informed tactical decisions.

### Opportunity

There is no existing tool tailored to the Trench Crusade dice mechanic. A purpose-built desktop app would let the user:
- Configure a dice scenario (pool size, pick top/bottom 2 or 3, modifier) and instantly see the probability distribution
- View cumulative probabilities (chance of achieving each result or higher) to inform tactical decisions during play

### Approach

Python + Tkinter desktop app. Tkinter is built into Python (zero dependencies), and the scope is well-defined for a V1.00.

#### Alternatives not chosen

- **PyQt/PySide** — More polished UI but adds a heavy dependency for a simple probability calculator. Overkill for V1.00.
- **Textual (TUI)** — Less intuitive for non-technical users; harder to display probability charts.
- **Web app** — User explicitly chose desktop.

#### Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| GUI framework | Tkinter | Built-in, no dependencies, sufficient for V1.00 |
| Dice type | Fixed d6 | Trench Crusade uses d6 exclusively |
| Pick count | User-configurable: 2 or 3 | TC mechanic uses 2 or 3 dice depending on situation |
| Pool size | User-configurable | Varies by action in the game |
| Modifiers | User-configurable (+/- integer) | Common in TC rules |
| Probability display | Cumulative probability chart | Shows chance of each result or higher |

## Goals

- Provide a desktop probability calculator implementing Trench Crusade's core d6 pool mechanic
- Allow user to configure pool size, top/bottom pick, pick count (2 or 3), and a single +/- modifier
- Display cumulative probabilities showing the chance of achieving each possible result or higher
- Keep the app simple, focused, and dependency-free

## Non-Goals

- Dice rolling / random simulation — this is a calculator, not a roller
- Exact (non-cumulative) probability display — future version
- Configurable dice types (d4, d8, d12, d20, etc.) — future version
- Pick counts other than 2 or 3 — future version
- Persistent storage / export of results — future version
- Network/multiplayer features
- Full game rules engine (wounds, saves, morale, etc.)

## Exit Criteria

- [ ] App launches as a desktop window via `python main.py`
- [ ] User can input: pool size (number of d6), top vs bottom selection, pick count (2 or 3), and a single +/- integer modifier
- [ ] App calculates and displays cumulative probabilities: the chance of achieving each possible result or higher
- [ ] Output is displayed as a chart or table within the app window
- [ ] Probabilities update when inputs change
- [ ] App handles edge cases: pool size equal to pick count (no selection needed), pool size of 1, modifier of 0
- [ ] Probability calculations are exact (combinatorial), not simulation-based
