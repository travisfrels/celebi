# V1.00 Trench Crusade Dice Roller

| Status | Set On |
|--------|--------|
| Planning | 2026-03-13 |
| Active | 2026-03-13 |

## Context

### Situation

The celebi repo is a new, empty repository with no source code. The user plays Trench Crusade (a tabletop miniatures wargame) and wants a dedicated desktop tool for rolling dice according to the game's core mechanic, with statistical tracking to understand outcome distributions.

### Opportunity

There is no existing tool tailored to the Trench Crusade dice mechanic. A purpose-built desktop app would let the user:
- Rapidly roll dice with proper TC rules (pool of d6s, pick top/bottom 2, apply modifiers)
- See accumulated roll results and their frequency distribution within a session

### Approach

Python + Tkinter desktop app. Tkinter is built into Python (zero dependencies), and the scope is well-defined for a V1.00.

#### Alternatives not chosen

- **PyQt/PySide** — More polished UI but adds a heavy dependency for a simple dice roller. Overkill for V1.00.
- **Textual (TUI)** — Less intuitive for non-technical users; harder to display frequency charts.
- **Web app** — User explicitly chose desktop.

#### Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| GUI framework | Tkinter | Built-in, no dependencies, sufficient for V1.00 |
| Dice type | Fixed d6 | Trench Crusade uses d6 exclusively |
| Pick count | Fixed: pick 2 | Core TC mechanic always sums 2 dice |
| Pool size | User-configurable | Varies by action in the game |
| Modifiers | User-configurable (+/- integer) | Common in TC rules |
| Frequency display | Bar chart or table in-app | Session-only, shows distribution of outcomes |
| Persistence | Session only | Resets on app close per user preference |

## Goals

- Provide a functional desktop dice roller implementing Trench Crusade's core d6 pool mechanic
- Allow user to configure pool size, top/bottom pick, and modifiers
- Track and display roll result frequency within a session
- Keep the app simple, focused, and dependency-free

## Non-Goals

- Configurable dice types (d4, d8, d12, d20, etc.) — future version
- Configurable pick count (always 2 for V1.00) — future version
- Persistent storage / export of roll history — future version
- Network/multiplayer features
- Full game rules engine (wounds, saves, morale, etc.)

## Exit Criteria

- [ ] App launches as a desktop window via `python main.py` (or similar)
- [ ] User can input: pool size (number of d6 to roll), top 2 vs bottom 2 selection, modifier value
- [ ] Clicking "Roll" rolls the dice pool, selects top/bottom 2, applies modifier, and displays the final result
- [ ] Individual dice values are shown so the user can see what was rolled
- [ ] Each roll result is appended to a session history list
- [ ] A frequency display (table or chart) shows how often each outcome has occurred in the session
- [ ] App handles edge cases: pool size of 2 (no selection needed), pool size of 1 (only 1 die available), modifier of 0
